# Verifying changes

The runtime surface is Storybook. Launch:

```bash
npm run storybook -- --no-open --ci   # port 6006, ready when GET / returns 200 (~10s)
```

Drive it headlessly with the repo's own Playwright (no separate install):

```js
import { chromium } from '<repo>/node_modules/playwright/index.mjs';
// story iframe (no manager chrome): http://localhost:6006/iframe.html?id=<title-slug>--<export-slug>
// e.g. primitives-poepanel--gallery, reference-reconstruction--dashboard
```

Gotchas / house rules:
- **Measure, don't eyeball**: judge placement/shading numerically — `getBoundingClientRect` in
  `page.evaluate`, then sample screenshot pixels with PIL (`np.array(Image.open(...))`) and compare
  row/column means. Screenshots are for the report; numbers are the verdict.
- Great A/B trick: screenshot, then flip a CSS var on the element via
  `el.style.setProperty('--var', ...)` in `page.evaluate`, screenshot again, diff rows.
- Visual fidelity itself is judged by the USER, not by us — verify mechanics (renders, measures,
  no regressions), don't declare fidelity.
- Rasters are generated by `src/components/primitives/*.sh` → `tools/framegen.py`; inspect PNGs
  numerically with PIL when a frame asset is in question.
